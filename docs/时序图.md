# 核心流程时序图

本文档描述系统两大核心流程的时序交互：**爬取流程** 与 **导出流程**。

## 1. 爬取流程时序图

涵盖从用户在 Web 界面点击「立即爬取」到结果写入数据库的完整链路。

```mermaid
sequenceDiagram
    actor User as 用户
    participant Web as Flask 路由<br/>(app.py)
    participant Thread as 后台线程
    participant Crawl as crawl_export.py
    participant Spider as Spider 实例<br/>(spiders/*.py)
    participant PW as Playwright<br/>Chromium
    participant Site as 政府网站
    participant KW as 关键词过滤器
    participant DB as MySQL

    User->>Web: POST /crawl_now（选择网站、时间范围）
    Web->>Thread: 启动后台线程（避免阻塞）
    Web-->>User: 立即返回 task_id
    Thread->>Crawl: run_crawl(websites, date_range)
    loop 每个启用的网站
        Crawl->>Spider: 实例化对应 Spider 类
        Spider->>PW: 启动无头 Chromium
        PW->>Site: 访问列表页（带反检测脚本）
        Site-->>PW: 返回 HTML（含瑞数 WAF）
        PW-->>Spider: 渲染后的页面
        Spider->>Spider: 解析文章列表
        loop 每条文章
            Spider->>KW: 匹配关键词
            alt 命中关键词
                Spider->>DB: 查询 url 是否已存在
                alt 不存在（去重通过）
                    Spider->>DB: INSERT articles
                else 已存在
                    Spider->>Spider: 跳过
                end
            else 未命中
                Spider->>Spider: 丢弃
            end
        end
        Spider->>PW: 关闭浏览器
        Spider->>Web: 上报进度（轮询接口）
    end
    Crawl-->>Thread: 完成
    User->>Web: GET /progress 查询进度
    Web-->>User: 返回实时进度（已爬/总数/命中数）
```

## 2. 导出流程时序图

用户在结果页筛选后点击导出，生成 Excel/CSV 文件下载。

```mermaid
sequenceDiagram
    actor User as 用户
    participant Web as Flask 路由<br/>(app.py)
    participant DB as MySQL
    participant Exp as Exporter<br/>(openpyxl / csv)
    participant FS as 文件系统<br/>output/

    User->>Web: GET /export?keywords=...&date_from=...&level=...
    Web->>DB: SELECT * FROM articles WHERE ...
    DB-->>Web: 文章结果集
    Web->>Exp: 生成 Excel/CSV
    Exp->>FS: 写入 output/政策监控_YYYYMMDD.xlsx
    FS-->>Exp: 文件路径
    Exp-->>Web: 文件流
    Web-->>User: 200 + Content-Disposition: attachment
    User->>User: 浏览器下载文件
```

## 3. 定时调度时序图（补充）

APScheduler 每 4 小时自动触发一次完整爬取。

```mermaid
sequenceDiagram
    participant Scheduler as APScheduler
    participant Crawl as crawl_export.py
    participant Spider as 各 Spider
    participant DB as MySQL

    Note over Scheduler: 每 4 小时触发
    Scheduler->>Crawl: trigger_crawl_all()
    Crawl->>Spider: 遍历所有 enabled=true 的网站
    Spider->>DB: 写入新文章
    Crawl-->>Scheduler: 完成，等待下次
```

## 4. 流程要点

| 流程 | 关键点 |
| --- | --- |
| 爬取 | 后台线程异步执行，避免阻塞 Flask 主线程；通过轮询 `/progress` 实现实时进度 |
| 反爬 | Playwright 注入反检测脚本绕过瑞数 WAF |
| 去重 | 以 `articles.url` 作为唯一键，数据库层面拦截重复 |
| 导出 | 支持按关键词、日期、级别筛选；生成文件保存到 `output/` 目录 |
| 调度 | APScheduler 每 4 小时自动跑一次全量；用户可随时手动触发 |
